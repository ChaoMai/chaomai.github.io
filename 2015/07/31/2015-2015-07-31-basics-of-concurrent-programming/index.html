Basics of Concurrent Programming

Posted on 2015-07-31   |   In programming   |  

The Amdahl’s Law

$$
S=\frac{1}{1-p+\frac{p}{n}}
$$

S：The speedup S of a job is the ratio between 一个处理器完成job的时间除以n个并发处理器完成job的时间。
p：job中，可以并行执行部分的比例。

假设一个处理器完成job的时间是1。有n个并发处理器，那么可并行执行的部分所用时间就是$p/n$，顺序执行部分所用时间是$1-p$，总的并行执行时间就是，

$$
1-p+\frac{p}{n}
$$

这个法则决定了，一个处理器完成一个job的时间，在增加n个处理器以后，时间并不是简单的变为原来的$1/n$。举个例子，10个房间，其中9个房间一样大，有个房间的大小是其他房间的2倍，有10个工人来刷漆。其中可并行的部分是$10/11$，带入公式以后，

$$
S=\frac{1}{1-\frac{10}{11}+\frac{\frac{10}{11}}{10}}=5.5
$$

可以增加到10个工人后，所用的时间其实只是1个工人的一半。

这点说明了，在系统中可能90%的代码都实现了并行，但是就是因为10%的必须要顺序执行，最后提升的效果就远不如预期。

Concurrent Objects

While all notions of correctness for concurrent objects are based on some notion of equivalence with sequential behavior, different notions are appropriate for different systems.

correctness conditions:

  • Quiescent consistency
  • Sequential consistency
  • Linearizability

Concurrency and Correctness

尽管并发的代码难以分析，但如果通过某种方法，把他们的并发执行map为顺序执行，那么分析并发对象就会容易些。

Sequential Objects

sequential specification

  • precondition: the object’s state
  • postcondition: the object’s state and return value
  • side effect: a change to an object’s state

for a queue,

  • 在操作前，队列的情况
  • 操作后，队列的情况以及所调用方法返回的值
  • 操作后，队列的情况

使用precondition和postcondition来定义object，对于单线程操作objects的时候是使用了，但是对于多个线程共享object的时候，这种specification就失效了。不仅仅是调用方法的时候，顺序是个问题（并发意味着调用可能会overlap，哪个先？）；object的state也是问题（单线程中，在方法调用之间，object的state必须是meaningful的，但是overlap导致了object不会处于方法调用之间这个状态）。

Quiescent Consistency

A method call is pending if its call event has occurred, but not its response event.

quiescent consistency

  1. Method calls should appear to happen in a one-at-a-time, sequential order.
    方法调用似乎是以一次一个，顺序的形式发生。
    但是这个条件并没有指明究竟是以什么样的sequential order来执行。

  2. Method calls separated by a period of quiescence should appear to take effect in their real-time order.
    被一段时间的quiescence分隔开的方法调用，似乎是以他们实时的顺序生效。

quiescent consistency表明了，如果在任意时刻，一个object是quiescent的，那么到目前为止的执行，等价于这些已完成的调用的某种顺序执行。

例如，假设A和B并发的把x和y enqueue（FIFO），接着queue变为quiescent，然后C再enqueue z。虽然不能够确定x和y在队列中的相对顺序，但是他们一定是在z前面的。

A method is total if it is defined for every object state; otherwise it is partial.

In any concurrent execution, for any pending invocation of a total method, there exists a quiescently consistent response.(in book’s 3.6)

A correctness property P is compositional if, whenever each object in the system satisfies P, the system as a whole satisfies P.

quiescent consistency is compositional, so quiescently consistent objects can be composed to construct more complex quiescently consistent objects.

Sequential Consistency

The order in which a single thread issues method calls is called its program order.
单线程程序调用方法时候的order。

sequential consistency

  1. Method calls should appear to happen in a one-at-a-time, sequential order.
    方法调用似乎是以一次一个，顺序的形式发生。

  2. Method calls should appear to take effect in program order.
    方法调用似乎是以程序规定的顺序的方式生效。

sequential consistency要求了方法调用，像they occurred in a sequential order consistent with program order一样的，起作用。

It is worth noting that sequential consistency and quiescent consistency are incomparable: Quiescent consistency does not necessarily preserve program order, and sequential consistency is unaffected by quiescent periods.

很多现代的CPU为了效率会进行reorder，默认并不会保证sequential consistency。如果需要sequential consistency，就必须显式的提出（使用memory barriers or fences）。

sequential consistency is not compositional.

Linearizability

Each method call should appear to take effect instantaneously at some moment between its invocation and response.
这陈述了方法调用时，必须保证实时行为。

Every linearizable execution is sequentially consistent, but not vice versa.

Linearization Points

  • for lock-based:
    each method’s critical section can serve as its linearization point.
    每个方法的临界区。

  • for implementations that do not use locking:
    the linearization point is typically a single step where the effects of the method call become visible to other method calls.
    一般是单独的一个步骤，这个步骤以后，effects对其他方法调用是可见的。

linearizability is compositional.

Sequential consistency适合描述一个独立的系统，比如硬件内存，composition不会是问题。Linearizability，相反，适合大系统的components，这些components必须被独立的验证。
